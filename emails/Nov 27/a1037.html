MUI + JSS

Mui.com est une librairie complète qui propose de nombreux composants :

À l'époque, MUI utilisait le JSS afin de créer des "custom class" :

Comme tu le vois c'est assez diabolique car il faut :

  1. Définir tes "classes" avec createUseStyles qui retourne un hook.
  2. Utiliser ce hook dans ton composant.
  3. Appliquer la bonne classe.

Mais bon... c'est très très verbeux.

Mais pourquoi utiliser le JSS ?

Bah, Mui nous propose des composants, mais ensuite tout le layout et autre élément custom, il faut quand même les créer soi-même.

C'est beaucoup plus simple et agréable à lire.

Mais bon... il y a quelques problèmes :

Tailwind

Puis un jour durant le live de Neolectron (oui oui) j'ai découvert Tailwind.

Un moyen de faire du CSS directement dans le HTML avec la prop className.

Elle va pouvoir gérer :

  • les pseudo-classes (hover etc...)
  • les thèmes
  • les plugins
  • le responsive
  • des centaines d'utilitaires (comme my-12 etc...)

Quand je l'ai découvert j'étais un peu déboussolé et perdu.

Mais après j'ai découvert son écosystème, des centaines de sites web qui proposent du code HTML à copier / coller qui juste... marche.

Pourquoi ?

Car Tailwind est une sorte de standard.

Ils proposent :

  • un thème de couleur
  • des tailles cohérentes
  • des shadows / borders / etc... bien foutues

Tout ça, avec une configuration qui ne prend même pas deux minutes.

Depuis le jour où je me suis rendu compte de tous ces avantages, c'est simple, je n'ai plus jamais utilisé autre chose que Tailwind.

ㅤ

Le piège...

Tailwind n'est pas une alternative à CSS, c'est une sorte de "super set".

Il faut non seulement maîtriser les "normes" Tailwind et tous les détails qui vont avec...

Mais aussi avoir une très bonne compréhension du CSS.

Car, contrairement au système de "grid" de Bootstrap, cette fois tout est simplement : du CSS.

Pour une bonne maîtrise de Tailwind, il faut donc maîtriser le CSS.

Mais je pense qu'une fois fait, tu as une rapidité d'exécution assez dingue, surtout avec toutes les super librairies qui participent à son écosystème :

  • Headless UI
  • Heroicons
  • Shadcn
  • Tailwind UI
  • FlowBit
  • HyperUI

ㅤ

Voilà pourquoi je suis addict à Tailwind.

Il m'offre la flexibilité du CSS avec la rapidité d'exécution de Tailwind.

C'est une sorte "d'inline style" mais optimisé, qui supporte les pseudo-classes, les thèmes, le responsive, etc...

Bref, une vraie dinguerie.
